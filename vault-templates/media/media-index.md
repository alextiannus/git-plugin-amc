## Storage Structure

```
media/
└── YYYY-MM-DD_{topic-slug}/
    ├── raw/        ← original files, never edited
    ├── edited/     ← final versions used in posts
    └── consent/    ← consent screenshots for customer/third-party media
```

**Naming rule:** `YYYY-MM-DD_{dish-or-topic}_{sequence}.jpg`
Example: `2026-05-01_malatangpot_01.jpg`

## Source Types

| Code | Meaning | Documentation required |
|---|---|---|
| `brand-owned` | Shot by brand or hired photographer | None |
| `licensed-stock` | Purchased from stock provider | License URL + purchase date in Notes |
| `customer-consent` | Customer photo with DM consent | Screenshot saved in consent/ folder |
| `third-party-consent` | Blogger/influencer photo with permission | Screenshot saved in consent/ folder |
| `ai-generated` | Created via mcp.image_gen | Requires mcp.image_gen enabled in SOUL.md |
