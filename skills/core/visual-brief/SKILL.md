# Skill · Visual Brief
# F&B Content Engine · core/visual-brief.md
# ─────────────────────────────────────────

## Purpose

每次创作内容，文案之外必须同时输出**视觉简报（Visual Brief）**。
视觉简报告诉品牌团队：**用哪个已有素材** 或 **需要拍什么新素材**。

没有视觉简报的内容草稿是不完整的。

---

## Step 0 · 素材检查（每次创作前必做）

在起草任何内容前，先检查素材库：

```
1. 读取 vault/media/media-index.md（如存在）
2. 检查是否有匹配当前内容主题的已审批素材
   - 匹配条件：同一产品 / 同一场景 / 同一情绪基调
3. 判断：
   A. 有合适素材 → 在 Visual Brief 中引用素材文件名，直接使用
   B. 素材过时（>90天）→ 标注"可用但建议重拍"
   C. 无素材 → 生成完整拍摄简报，发给团队
```

## Visual Brief 格式

**每个平台的内容草稿之后，紧接输出以下格式：**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📷 VISUAL BRIEF · [平台名] · [内容标题]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
素材状态: [✅ 使用已有素材 / 🎬 需要新拍摄 / ⚠️ 现有素材可用但建议更新]

[如果使用已有素材]
使用文件: [文件名 / Lark Drive 链接]
使用说明: [用哪几张、顺序建议]

[如果需要新拍摄]
主视觉 Hero Shot:
  被摄主体: [具体描述 — 哪个产品、哪个角度、特写还是全景]
  场景/背景: [桌面、厨房、街道、白底、品牌空间等]
  光线: [自然光 / 柔光棚拍 / 暖色室内灯 / 户外自然光]
  道具: [餐具、配料、包装、无道具等]
  人物: [纯产品 / 入镜手部 / 入镜半身 / 无人]
  情绪基调: [从品牌视觉风格中提取，如"温暖、接地气"或"简洁、高级"]

配图/辅助素材:
  需要数量: [X 张]
  内容描述: [每张图的具体内容，例如：制作过程、特写细节、用餐场景]

文字叠加 (Text Overlay):
  [是否需要 / 叠加什么文字 / 哪一张上叠加]

平台规格:
  格式: [见下方规格表]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 平台视觉规格速查

| 平台 | 主格式 | 尺寸/比例 | 视频时长 | 关键要求 |
|---|---|---|---|---|
| **Instagram** | 轮播图 | 1:1 或 4:5 | — | 第1张必须是最强钩子图 |
| **Instagram Story** | 竖版单图/视频 | 9:16 | ≤60s | 上下留白给互动贴纸 |
| **TikTok** | 竖版视频 | 9:16 | 15–60s | 前2秒必须有动作/声音 |
| **小红书** | 轮播图 | 3:4 或 1:1 | — | 封面字体要大，标题可见 |
| **小红书视频** | 竖版视频 | 9:16 | 30s–3min | 字幕必须 |
| **Facebook** | 横版图/视频 | 16:9 或 1:1 | ≤3min | 文字不超过图片20% |
| **YouTube** | 横版视频 | 16:9 | 8–15min | 缩略图文字≤5字，高对比 |
| **X / Twitter** | 横版图 | 16:9 | ≤2min 20s | 图文结合优于纯图 |

## 品牌视觉风格注入

**在生成每份 Visual Brief 前，必须从 `brand-voice.md` 读取以下字段并应用：**

```
- BRAND_VISUAL_STYLE: [品牌视觉风格关键词]
- BRAND_COLORS: [品牌主色调]
- VISUAL_REFERENCE: [参考账号或 mood board 链接]
- ASSETS_LOCATION: [素材库 Lark Drive 路径]
```

**这些字段决定：**
- 情绪基调选词（用 Visual Style 的词汇）
- 道具/背景颜色建议（品牌色优先）
- 拍摄风格参照（Reference 账号的构图/光线风格）

## 素材需求升级决策树

```
内容创作时 →

是否有匹配素材？
├─ YES → 直接引用 → 在 Kanban 任务标注"素材已就位"
└─ NO →
    是否是常规内容（产品图、餐饮场景）？
    ├─ YES → 输出拍摄简报 → 发 Lark 通知团队"需要新拍摄"
    └─ NO（故事类、活动类、KOL 合作）→
        输出拍摄简报 + 标注"高优先级" → Kanban 创建 pending 任务等待确认
```

## 拍摄简报 Lark 通知模板

当检测到需要新拍摄时，自动发送以下 Lark 消息：

```
📷 新素材需求 / New Visual Assets Needed

内容: [内容标题]
平台: [目标平台]
计划发布: [日期]

需要拍摄:
[从 Visual Brief 提取的核心拍摄说明]

建议在 [发布日期 - 3天] 前完成拍摄并上传到：
[ASSETS_LOCATION]

如果无法及时拍摄，请回复"跳过"——我会改用已有素材或文字图替代。
```

## 与 Repurpose Chain 的集成

在 `repurpose-chain` 的每个平台步骤完成后：

```
Step 1 Instagram 文案 → 输出 Visual Brief (Instagram 规格)
Step 2 TikTok 脚本   → 输出 Visual Brief (TikTok 规格，含分镜指引)
Step 3 小红书文案    → 输出 Visual Brief (小红书规格)
Step 4 Facebook 文案 → 复用 Instagram 素材（标注），或输出独立 Brief
Step 5 YouTube 脚本  → 输出 Visual Brief (YouTube 规格，含封面简报)
Step 6 X/Twitter     → 复用已有素材（标注），文字图可自动生成
```

TikTok 和 YouTube 的 Visual Brief 还需包含**分镜表（Shot List）**：

```
分镜表 Shot List:
[0:00–0:02] 镜头1: [具体拍什么 — 例：筷子夹起饺子，露出汤汁拉丝]
[0:02–0:08] 镜头2: [例：厨师下锅，蒸汽升起]
[0:08–0:18] 镜头3: [例：成品全景，手拿菜单指向价格]
[0:18–0:30] 镜头4: [例：顾客入镜表情，点头满意]
```
