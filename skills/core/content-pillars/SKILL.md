# Skill · Content Pillars
# F&B Content Engine · core/content-pillars.md
# ─────────────────────────────────────────────

## Purpose

每条内容都属于某一个支柱。
支柱决定：**这条内容在整体品牌叙事中扮演什么角色**。

没有支柱框架，内容只是一堆孤立的帖子。
有了支柱框架，内容变成一个有节奏、有层次的品牌故事。

---

## 五大内容支柱

### Pillar 1 · 产品聚焦 Product Spotlight
**默认比例：40%**

> 目标：让目标受众看到产品、产生渴望、采取行动（购买/预订/到店）

| 属性 | 说明 |
|---|---|
| 内容重点 | 单个产品/服务的展示、价格、卖点、感官细节 |
| 推荐 Hook | craving / deal / discovery |
| 推荐格式 | 轮播图（产品细节） / 短视频（制作过程） / 单图（价格钩子） |
| 适合平台 | Instagram ★★★ · TikTok ★★★ · 小红书 ★★★ · Facebook ★★ |
| 禁忌 | 过于频繁的促销语气；连续 3 条以上不间断产品推广 |

**内容创作提示：**
- 每条必须有一个具体数字（价格/份量/时间/次数）
- 避免"美味"/"好吃"等无意义形容词
- 图片/视频必须展示产品本身，不能只有文字

---

### Pillar 2 · 品牌故事 Brand Story
**默认比例：25%**

> 目标：建立信任、传递品牌价值观、让用户产生情感连接

| 属性 | 说明 |
|---|---|
| 内容重点 | 幕后故事、创始人/团队/师傅、品牌历史、选材过程、品牌价值观 |
| 推荐 Hook | bts（幕后） / community（人情味） |
| 推荐格式 | 短视频（人物故事） / 轮播图（时间轴/过程展示） |
| 适合平台 | TikTok ★★★ · YouTube ★★★ · 小红书 ★★★ · Instagram ★★ |
| 禁忌 | 过于企业化的语气；公关稿式的表达 |

**内容创作提示：**
- 必须有真实人物或真实场景（不能是抽象的"我们品牌相信..."）
- 用第一人称或第三人称具体人物叙述
- 允许"不完美"——真实的制作失误、学习过程比精美更有说服力

---

### Pillar 3 · 知识教育 Educational
**默认比例：15%**

> 目标：建立权威地位、提升内容分享率、吸引主动搜索的新用户

| 属性 | 说明 |
|---|---|
| 内容重点 | 行业知识、制作技巧、选购指南、文化背景、常见误区纠正 |
| 推荐 Hook | discovery（发现） / contrarian（颠覆认知） |
| 推荐格式 | 轮播图（step-by-step 教程） / 短视频（演示） / YouTube（深度） |
| 适合平台 | 小红书 ★★★ · YouTube ★★★ · Instagram ★★★ · X ★★ |
| 禁忌 | 过于学术化；内容与品牌产品脱节 |

**内容创作提示：**
- 每条教育内容必须和品牌产品有自然关联（不能是纯科普）
- 结尾必须有 soft CTA 引导到品牌产品
- 标题要像"你不知道的 X 件事"而非"关于 X 的介绍"

---

### Pillar 4 · 社群互动 Community
**默认比例：12%**

> 目标：提升互动率、建立归属感、激活粉丝成为品牌传播者

| 属性 | 说明 |
|---|---|
| 内容重点 | 顾客故事、UGC 转发、问答互动、投票、节日祝福、里程碑庆祝 |
| 推荐 Hook | community（社群认同） |
| 推荐格式 | 图文（引发评论） / Story 互动贴纸 / Facebook 帖子+问题 |
| 适合平台 | Facebook ★★★ · Instagram Story ★★★ · 小红书 ★★ |
| 禁忌 | 假互动（"双击如果你..."）；忽视评论区回复 |

**内容创作提示：**
- 每条必须有明确的互动邀请（提问/投票/标记好友）
- UGC 转发时必须 @原作者并致谢
- 避免节日帖子流于形式，必须连接品牌产品或故事

---

### Pillar 5 · 促销转化 Promotional
**默认比例：8%**

> 目标：直接驱动销售/预订行动，配合特定优惠或限时活动

| 属性 | 说明 |
|---|---|
| 内容重点 | 限时折扣、新品上市、套餐促销、节假日特供 |
| 推荐 Hook | deal（价格钩子） |
| 推荐格式 | 单图（清晰价格+CTA） / Story（限时倒计时） |
| 适合平台 | Instagram ★★★ · Facebook ★★★ · 小红书 ★★ |
| 禁忌 | 连续发布 2 条以上促销（触发用户屏蔽）；价格信息不准确 |

**内容创作提示：**
- 价格、日期、库存限制必须 100% 准确（FTC 合规）
- 促销帖之后必须至少间隔 2 条非促销内容
- 促销结束后立即关闭或更新帖子

## 支柱比例管理

### 默认比例（品牌通用）

```
Product Spotlight:  40%
Brand Story:        25%
Educational:        15%
Community:          12%
Promotional:         8%
```

### 比例执行规则

**周发帖量对应的支柱分配（以每周 15 条为例）：**

| 支柱 | 15条/周 | 10条/周 | 5条/周 |
|---|---|---|---|
| Product Spotlight | 6 条 | 4 条 | 2 条 |
| Brand Story | 4 条 | 2 条 | 1 条 |
| Educational | 2 条 | 2 条 | 1 条 |
| Community | 2 条 | 1 条 | 1 条 |
| Promotional | 1 条 | 1 条 | 0 条 |

**比例调整触发条件（来自 performance-learning）：**

```
若某支柱内容的平均互动率连续 4 周高于其他支柱 >30%：
→ 将该支柱比例上调 5%，从最低比例支柱扣除
→ 更新 vault/brandcontext.md 中的内容支柱比例
→ Lark 通知："[支柱名] 内容表现持续领先，已自动上调比例至 [N]%"
```

### 节点内容原则

```
1. 节点内容必须连接品牌产品（不能是纯祝福图）
2. 节点促销只能占节点内容的 50%（另 50% 是故事/文化/情感）
3. 提前备稿：节点前 [N] 天在 Content Schedule Bitable 创建 status=ready 条目
4. 节点后 48h：发一条总结/感谢内容（Community 支柱）
```

## Campaign 管理

### 什么是 Campaign

Campaign = 围绕一个主题的多条内容，形成有起承转合的叙事弧度。
**单条内容 ≠ Campaign。3–8 条内容 + 统一主题 = Campaign。**

### Campaign 四幕结构

```
Act 1 · 预热 Teaser（Campaign 前 3-5 天）
  目标：引发好奇，不揭示全貌
  支柱：Brand Story 或 Educational
  Hook：discovery / contrarian
  例："有件事我们准备了很久，这周揭晓。"

Act 2 · 发布 Launch（Campaign 第 1 天）
  目标：正式亮相，激发行动
  支柱：Product Spotlight + Promotional（如有）
  Hook：deal / craving
  例：完整产品/活动介绍，价格/时间清晰可见

Act 3 · 持续 Sustain（Campaign 期间，每 2-3 天）
  目标：维持热度，多角度深化
  支柱：轮换（Educational → Community → Brand Story）
  策略：每条从不同角度切入同一主题，避免重复
  例：第1条讲制作过程；第2条讲顾客故事；第3条讲文化背景

Act 4 · 收尾 Recap（Campaign 最后一天）
  目标：感谢参与、沉淀成果、引导下一步
  支柱：Community
  例："感谢这周所有来过的朋友，下一个惊喜..."
```

### Campaign 创建命令

团队发送以下 Lark 消息即可触发 Campaign 规划：

```
发起Campaign: [主题名称] / [开始日期] / [持续天数] / [核心产品]
例: 发起Campaign: 夏日限定冷饮 / 2025-06-01 / 14天 / 冷萃咖啡+果茶
```

Agent 收到后：
```
1. 规划四幕结构，计算每幕发帖时间
2. 为每条内容生成主题草案（不是完整文案，是方向+支柱+hook类型）
3. 在 Content Schedule Bitable 创建所有 Campaign 条目，status=draft
4. 发 Lark 确认："Campaign [名称] 已规划，共 [N] 条内容，请确认方向后我开始创作。"
```

## 与其他系统的集成

### → Topic Discovery（08:00 cron）

每天主题发现时，必须检查支柱缺口：

```
1. 读取本周已发布/已排期内容，统计各支柱占比
2. 与目标比例对比，识别缺口支柱
3. 优先为缺口支柱生成新内容创意
4. 在提案前标注："[Educational 缺口] 本周 Educational 内容为 0，
   这条创意优先补充该支柱"
```

### → Repurpose Chain

每次运行 Repurpose Chain 时，Step 0 决策输出必须包含：

```
[Pillar] 本条内容归属：[支柱名] | 本周 [支柱名] 比例：[当前%] → 目标 [目标%]
```

### → Performance Learning

每月支柱比例分析：

```
对比各支柱平均互动率 → 自动调整比例 → 更新 vault/brandcontext.md（内容支柱章节）
```

### → Weekly Report

每周报告中必须包含支柱健康度检查：

```
支柱分布本周实际 vs 目标：
- Product Spotlight: 实际 45% / 目标 40% [+5%，偏重]
- Brand Story:       实际 15% / 目标 25% [-10%，不足 ⚠️]
- Educational:       实际 20% / 目标 15% [+5%，略超]
- Community:         实际 15% / 目标 12% [基本达标]
- Promotional:       实际  5% / 目标  8% [略低]

建议：下周增加 2 条 Brand Story 内容补足缺口。
```
